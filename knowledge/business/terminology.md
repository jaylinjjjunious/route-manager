# Domain Terminology

| Term | Definition |
|------|-----------|
| **Shower Gate** | Daily barcode verification system that must be completed before work begins. |
| **Cycle** | 24-hour period between daily resets (resets at 6:00 AM local time). |
| **Ride Mode** | Distraction-free execution surface for completing jobs during active route. |
| **Proof Vault** | Storage for job completion attachments (photos, screenshots, receipts). |
| **AI Dispatcher** | Gemini-powered chat assistant that provides route and job advice. |
| **Safety Brief** | Area-specific crime and safety news for the work area. |
| **Outlier** | A job flagged as an inefficient route addition (more than 4.2 miles from nearest neighbor). |
| **Route A** | The main optimized job sequence for the current session. |
| **Route B** | Standby jobs not yet assigned to the active route. |
| **Process Serve** | Legal document delivery job type, tracked as a first-class job type. |
| **Battery-Aware Routing** | Route optimization that factors in e-bike battery range and consumption. |
| **Mission Control** | The Dashboard screen showing shower gate status and ride mode controls. |
| **Checkpoint** | A stable annotated Git tag used for safe rollback to a known-good state. |
| **EndOfDaySummary** | Summary view shown in the Tracker tab after ending Ride Mode. |
| **Habit Streak** | Consecutive days of completing all mandatory habits (shower + custom tasks). |
| **Job Type** | Classification of a job: default, revision, process_serve, delivery, pickup. |
| **Job Status** | Lifecycle state: pending, in_progress, completed, failed, skipped, revision, archived. |
